# WCAG 2.2 AA + paridad i18n completa de la web (@kw/web) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Llevar la SPA React `@kw/web` a **conformidad WCAG 2.2 AA** y a **paridad i18n completa**. Dos frentes: (1) reconciliar los 5 locales (`en`, `es`, `de`, `pl`, `pt_BR`) al **mismo conjunto de claves** que `en/translation.json` y envolver en `t()` **todo** texto visible que el restyle dejó hardcodeado en inglés; (2) accesibilidad: `alt` significativo en imágenes (retratos, logo), `aria-label` en botones de solo icono, controles de formulario asociados a `<label>` (primitivo `Field`), jerarquía de encabezados correcta (un `h1` por vista, sin saltos), foco visible, **modales con foco atrapado + Escape + `aria-modal` + etiquetado**, regiones `aria-live` para notificaciones de tirada de dados, y respeto de `prefers-reduced-motion`. **Sin tocar lógica de negocio, datos, hooks, llamadas a API ni rutas** — solo marcado, accesibilidad e i18n. Tras cada tarea el paquete compila (`typecheck`) y los tests de web siguen verdes.

**Architecture:** `@kw/web` es el adaptador de UI del monorepo hexagonal. Esta fase **no añade ni modifica** datos, hooks (`useSession`, `useCharacters`, `useParties`, `useInventory`, `useGenerators`, `useDiceRoller`), capa `src/api/*` ni `src/realtime/*`, ni las rutas de `App.tsx`. Se opera sobre la capa de presentación: primitivos `src/ui/` (`Modal`, `Field`, `Spinner`, `Button`), layout `src/layout/`, páginas de cada área (`auth`, `characters`, `parties`, `inventory`, `generators`) y los 5 ficheros de locales en `src/i18n/locales/*/translation.json`. El `LanguageSelector` conserva su `<select id="lang-selector">` con sus 5 `<option>` y su `aria-label` (los tests lo consultan por rol/etiqueta); el `Modal` conserva `role="dialog"` + `title` como `heading` (los tests lo consultan por `getByRole`).

**Tech Stack:** Node 22, pnpm 11, TypeScript 5, React 18, Vite 5, Tailwind CSS v4, `react-i18next` / `i18next`, Vitest + Testing Library (jsdom). Paquete: `@kw/web`. Entorno Windows.

> **Convención i18n (DURA):** la **clave es el texto en inglés** (identidad en `en`). Al añadir una clave hay que ponerla en **LOS 5 ficheros** `src/i18n/locales/{en,es,de,pl,pt_BR}/translation.json` con su traducción (`en`=identidad; `es` de alta calidad; `de`/`pl`/`pt_BR` best-effort coherente). Los 5 ficheros deben acabar **con el mismo conjunto de claves**. Todo texto visible pasa por `t()`. **No** se renombran claves existentes.

> **Convención A11y (DURA):** objetivo WCAG 2.2 AA. **No** se baja el contraste ya corregido (tokens `accent`/`btn`/`danger` en `index.css`). **No** se rompen roles/labels existentes que los tests consultan (`getByRole`, `LanguageSelector`, `Modal` heading). Imágenes con `alt` significativo, botones de solo icono con `aria-label`, controles asociados a `<label>` vía `Field`, jerarquía de encabezados correcta, foco visible, modales con foco atrapado + Escape + `aria-modal` + etiquetado, `aria-live` para tiradas, `prefers-reduced-motion`.

**Accesibilidad — hallazgos:**

- `ui/Modal.tsx`: tiene `aria-modal`, `role="dialog"`, Escape y `aria-labelledby` condicional, **pero NO atrapa el foco** ni mueve el foco al abrir ni lo restaura al cerrar (WCAG 2.4.3, 2.1.2). Falta `aria-label` de respaldo cuando no hay `title`.
- `layout/NavDrawer.tsx`: igual, sin foco atrapado/restaurado (es un `dialog`).
- `ui/Spinner.tsx`: `aria-label="Loading"` hardcodeado en inglés (debería usar `t()`; pero es primitivo sin hook — se traducirá vía prop o `useTranslation` interno).
- `i18n/LanguageSelector.tsx`: `aria-label="Select language"` hardcodeado.
- `characters/PrintCharacterPage.tsx`: `alt="character portrait"` no descriptivo y hardcodeado; además **orden de encabezados roto** (`<h3>` "Inventory" en línea 26 aparece antes del `<h1>` del nombre en línea 100 dentro del flujo del documento de impresión — el `h1` debe ser el primero).
- `generators/ToolsPage.tsx`: el título de página es `<h2>` (línea 23); debería ser `<h1>` (un h1 por vista).
- `characters/ImportCharacterPage.tsx`: `<label>` (línea 63) sin `htmlFor` asociado al `<input type="file">` (línea 66) — no es un control etiquetado.
- `generators/DiceModal.tsx`: `<Field label="Count" htmlFor="dice-count">` ya asocia; pero los labels `"Count"`/`"Die"` no pasan por `t()`.
- `index.css`: **no** hay regla `prefers-reduced-motion` que desactive animaciones (`animate-spin` del Spinner, `transition-colors` de Button). WCAG 2.3.3 (AAA) / buena práctica AA.
- `aria-live`: `PartyViewPage` ya tiene `aria-live="polite"` en la `Card` de tiradas (línea 87). Correcto; mantener.

> **Nota de diseño abierta:** `parties/PartyViewPage.tsx` muestra `"Character #{memberId}"` como literal técnico (no es copy traducible real, es un placeholder de id). Se trata como cadena traducible con interpolación `t("Character #{{id}}", { id })` para no dejar inglés visible, **sin** cambiar la lógica. Si el diseño prefiere ocultarlo, queda fuera de alcance (no se toca lógica/datos).

```tsx
import { useEffect, useId, useRef, type ReactNode } from "react";
import { useTranslation } from "react-i18next";
import { cn } from "./cn.js";

export interface ModalProps {
  open: boolean;
  onClose: () => void;
  title?: ReactNode;
  children: ReactNode;
  className?: string;
}

const FOCUSABLE =
  'a[href],button:not([disabled]),textarea,input,select,[tabindex]:not([tabindex="-1"])';

export function Modal({ open, onClose, title, children, className }: ModalProps) {
  const { t } = useTranslation();
  const titleId = useId();
  const dialogRef = useRef<HTMLDivElement>(null);
  const previousFocus = useRef<HTMLElement | null>(null);

  useEffect(() => {
    if (!open) return;
    previousFocus.current = document.activeElement as HTMLElement | null;
    const dialog = dialogRef.current;
    const focusables = dialog?.querySelectorAll<HTMLElement>(FOCUSABLE);
    (focusables && focusables.length ? focusables[0] : dialog)?.focus();

    const onKey = (e: KeyboardEvent) => {
      if (e.key === "Escape") {
        onClose();
        return;
      }
      if (e.key !== "Tab" || !dialog) return;
      const items = dialog.querySelectorAll<HTMLElement>(FOCUSABLE);
      if (items.length === 0) {
        e.preventDefault();
        return;
      }
      const first = items[0];
      const last = items[items.length - 1];
      const active = document.activeElement;
      if (e.shiftKey && active === first) {
        e.preventDefault();
        last.focus();
      } else if (!e.shiftKey && active === last) {
        e.preventDefault();
        first.focus();
      }
    };
    document.addEventListener("keydown", onKey);
    return () => {
      document.removeEventListener("keydown", onKey);
      previousFocus.current?.focus?.();
    };
  }, [open, onClose]);

  if (!open) return null;

  return (
    <div
      className="fixed inset-0 z-50 flex items-center justify-center bg-black/50 p-4"
      onClick={onClose}
    >
      <div
        ref={dialogRef}
        role="dialog"
        aria-modal="true"
        aria-labelledby={title ? titleId : undefined}
        aria-label={title ? undefined : t("Dice roller")}
        tabIndex={-1}
        className={cn(
          "max-h-[90vh] w-full max-w-lg overflow-y-auto rounded-lg border border-border bg-surface p-6 shadow-lg focus:outline-none",
          className
        )}
        onClick={(e) => e.stopPropagation()}
      >
        {title ? (
          <h2 id={titleId} className="mb-4 font-serif text-xl text-text">
            {title}
          </h2>
        ) : null}
        {children}
      </div>
    </div>
  );
}
```
